===========================================================
    턴제 전략 카드 배틀 게임 (V2) - 시스템 구조 및 코드 명세서
===========================================================

본 문서는 현재 복원된(AP 즉발형이 아닌 예약 큐 기반) 3x3 턴제 카드 배틀 게임의 전체 시스템 동작 원리와 각 파일이 수행하는 역할을 상세하게 설명합니다.

-----------------------------------------------------------
1. 게임의 핵심 동작 원리 (Game Engine Flow)
-----------------------------------------------------------
이 게임은 실시간이 아닌 '동시 턴 입력 & 순차 교차 실행(Simultaneous Turn & Interleaved Resolution)' 방식을 따릅니다.
두 플레이어(나와 적)는 각자의 턴 페이즈에서 3개의 행동(Move, Card, Draw, Rest 등)을 미리 계획(Queue)합니다.
양 플레이어가 모두 준비를 완료(Ready)하면 [실행 페이즈(Resolving)]로 넘어가 등록해둔 3개의 행동을 번갈아 가며 1개씩 총 6단계(Step 0~5)에 걸쳐 실행합니다.
이 과정에서 상태이상, 버프, 데미지 등은 `statusEngine`과 `skillEngine`을 통해 처리 및 중재됩니다.


-----------------------------------------------------------
2. 주요 파일별 상세 코드 설명 및 역할
-----------------------------------------------------------

[ A. src/types/index.ts - 데이터/타입 정의 ]
이 파일은 프로젝트 전반에서 사용되는 TypeScript 인터페이스와 공통 타입을 지정합니다.
1) Card / CardEffect 관련
   - CardType, CardPriority, EffectTrigger 등 텍스트 JSON의 데이터를 파싱받을 그릇을 정의합니다.
   - 각 카드는 사거리(range), 데미지(damage), 대상(target)을 가지며 내부적으로 effects 배열을 통해 기절, 중독, 회복 등을 트리거합니다.
2) PlayerState / GameState 
   - PlayerState에는 체력(hp), 기력(energy), 덱(deck), 손패(hand), 묘지(discardPile), 상태이상 목록(statuses), 그리고 ⭐ actionQueue(길이 3의 배열)와 isReady(준비 스위치) ⭐ 상태가 존재합니다.
   - GameState의 status는 게임의 흐름을 나타냅니다. ('lobby' -> 'planning' -> 'resolving' -> 'discarding' 등)

[ B. src/engine/store.ts - 핵심 무결성 상태 관리 (Zustand) ]
React 생태계에서 전역 상태를 관리하는 Zustand Store이며, 실질적인 "게임 서버(엔진)" 역할을 수행하는 파일입니다.
1) initGame()
   - 양측 플레이어의 덱을 생성하고, 플레이어1은 하단 중앙(0, 2), 플레이어2는 상단 중앙(2, 0) 좌표에 스폰시킵니다.
   - P2(적 AI)의 행동은 `simulateAI` 함수를 통해 자동으로 난수를 돌려 3개의 큐를 채웁니다.
2) stageAction() / appendAction() / unstageAction()
   - 유저가 화면에서 행동을 1개, 2개, 3개 씩 채워 넣거나 뺄 때 쓰입니다.
   - 만약 카드 사용이라면 카드의 비용(cost)이 보유한 기력(energy)보다 많은지 이 단계에서 체크하여 스킵하거나 예약합니다.
3) setReady()
   - 양측이 슬롯 3개를 모두 채우거나 턴을 넘길 때 발동됩니다. (P1 & P2 isReady === true 조건 달성 시 'resolving' 흐름으로 전환)
4) resolveNextStep() (가장 중요한 핵심 실행기)
   - 0~5 스텝동안 setInterval 혹은 setTimeout 방식으로 주기적으로 호출됩니다.
   - step 0: P1의 첫 행동 -> step 1: P2의 첫 행동 -> step 2: P1의 두 번째 행동 -> ...
   - 기절이나 빙결 상태라면 행동은 무시(취소)됩니다.
   - 플레이 카드(`play_card`)일 경우 덱에서 꺼낸 뒤 StatusEngine과 SkillEngine에 데미지/회복/상태이상 부착 로직을 돌립니다. (가속 크리티컬 등의 히든 기믹 포함)
   - 6단계가 끝나면 다시 'planning' 상태로 돌아가거나, 손패가 10장이 넘는지 검사하여 'discarding' 상태로 넘깁니다.

[ C. src/engine/statusEngine.ts - 상태이상 및 패시브 시스템 ]
각종 디버프(독, 기절, 침묵, 동결 등)나 버프(보호막 등)를 처리합니다.
- on_turn_start / on_turn_end 에 맞춰 도트 딜을 넣거나 (poison: 체력-10) 버프 지속 턴 수를 차감합니다.
- getDamageMultiplier() 등의 함수는 약점 노출 상태인 적을 칠 때 피해량을 30~50% 뻥튀기하는 계수를 제공합니다.

[ D. src/engine/skillEngine.ts - 고유/특수 기믹 엔진 ]
캐릭터 고유의 스킬 기믹을 하드코딩 또는 동적 검사로 주입하는 곳입니다.
- 예를 들어, 특정 캐릭터가 보유한 Blink(점멸) 카드를 썼다면 원래 이동 시스템을 무시하고 해당 타겟으로 즉시 점프하는 overrideMove() 로직을 가로챕니다.
- onCollision(충돌 로직), getExtraAttacks(연타 횟수 추가) 등 기본 시스템에 예외를 두어 게임의 확장성을 높입니다.

[ E. src/app/game/page.tsx - 메인 UI 렌더링 화면 ]
모든 데이터를 브라우저에 그려내고, 사용자 입력을 받아 store.ts에 전달하는 "프론트 컨트롤러"입니다.
1) 행동 메뉴(ActionPanel) / 마우스 보드(FieldBoard) 조작
   - 우측 화면하단에 있는 버튼들(Move, Card, Draw)을 눌러서 현재 모드(currentCommand)를 바꿉니다.
   - 그 후 중앙의 3x3 보드판(FieldBoard)의 타일을 클릭하면 store의 `appendAction(...)` 을 호출하여 내 행동 큐에 쌓습니다.
2) 하단(HandSystem) 및 손패 렌더링
   - 자신의 패에 들어있는 카드 일러스트를 부채꼴로 쏴줍니다. 카드를 누르면 타겟팅 모드로 진입합니다.
3) Action Queue(예약 슬롯 3개) 표시 / Turn End 버튼
   - 화면 우측에 [아이콘-아이콘-아이콘] 3개가 그려진 칸이 있으며, 내가 등록한 행동이 순서대로 뜹니다.
   - 클릭하면 행동이 지워집니다(`unstageAction`). 3개가 모두 차면 빨간 Turn End 버튼을 누를 수 있게 활성화됩니다(`setReady`).
4) 화려한 전투 로그 & 이펙트 관리
   - store.logs 배열을 감지하여 타이핑된 로그들을 화면 좌상단에 스크롤로 뿌려줍니다.
   - store.status 가 'resolving'일 땐 중앙에 크게 "BATTLE PHASE"라는 글씨를 단 몇 초간 띄우고 지웁니다.

[ F. src/components/game/ui/* 폴더 컴포넌트들 ]
1) PlayerStatus.tsx
   - 상/하단에 위치한 플레이어의 HP, EP 게이지바 그라데이션과 덱/무덤 매수를 표시합니다.
   - 체력이 깎일 시 애니메이션(데미지 팝업, 붉은 번쩍임)을 직접 CSS와 useState로 관리합니다.
2) FieldBoard.tsx
   - 3x3 그리드 보드입니다. 카드를 타겟팅할 때 어디까지 닿을 수 있는지(사거리: line, adjacent 등) 파악해 붉게 혹은 노랗게 타일에 빛을 뿌려주는 프리뷰(PreviewTiles) 역할을 함께합니다.
3) Card.tsx (GameCard3D)
   - Framer Motion을 이용하여 마우스를 올렸을 때 카드가 3D로 기울어지거나 빛이 감도는(글레어 이펙트) 화려한 프리미엄 AAA 애니메이션을 전담합니다.

-----------------------------------------------------------
결론 및 유지보수 가이드
-----------------------------------------------------------
이 시스템은 "상태(데이터)"와 "뷰(렌더링)"가 완벽하게 분리되어 있습니다.
- 로직 수정이나 새 카드 효과 추가: `store.ts` 및 `engine/*.ts` 수정
- UI 디자인 변경, 카드 이펙트 추가: `page.tsx` 및 `components/game/ui/*.tsx` 수정
- 큐 시스템 자체를 바꾸려면 `page.tsx`의 actionQueue UI와 `store.ts` 의 resolveNextStep 6단계 핑퐁 파트를 동시에수정해야 합니다.
